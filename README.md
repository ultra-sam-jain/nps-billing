# Power Solutions Hub - UPS & Battery Calculator

Power Solutions Hub is a browser-based UPS, battery, and accessory quotation calculator. It lets a user filter compatible products, add items to a quote, review pricing with GST, capture customer details, and generate a printable proposal/PDF.

The project is currently implemented as a self-contained single-page application in `index.html`.

## Project Structure

```text
battery calculator/
|-- index.html   # Complete application: markup, styles, data, and JavaScript logic
`-- README.md    # Project documentation
```

There is no build system, package manager, backend, database, or framework dependency required for normal use.

## How To Run

Open the app directly in a browser:

```text
D:\battery calculator\index.html
```

Or serve it locally:

```powershell
cd "D:\battery calculator"
python -m http.server 8765 --bind 127.0.0.1
```

Then open:

```text
http://127.0.0.1:8765/index.html
```

## Main Features

- UPS product filtering by brand, kVA capacity, phase, and VDC.
- Battery filtering by type, brand, and AH rating.
- Accessory browsing by category.
- Cart-style quote builder.
- Automatic subtotal, GST, and grand total calculation.
- Proposal modal with customer details.
- Printable quotation/PDF layout.
- Copy-to-clipboard proposal text.
- Responsive layout for desktop and smaller screens.

## Data Model

### UPS Products

UPS records are stored in `UPS_DATA`.

Each item includes:

```js
{
  id: Number,
  name: String,
  rate: Number,
  warranty: String
}
```

The app derives additional fields from the product name:

- `brand`
- `kva`
- `vdc`
- `phase`
- `category`

These are generated by helper functions such as:

- `extractBrand`
- `extractKVA`
- `extractVDC`
- `extractPhase`
- `enrichUPS`

### Battery Products

Battery records are stored in `BATTERY_DATA`.

Derived fields include:

- `brand`
- `type`
- `ah`
- `voltage`
- `category`

These are generated by:

- `extractBatteryType`
- `extractAH`
- `extractVoltage`
- `enrichBattery`

### Accessories

Accessory records are stored in `ACCESSORY_DATA`.

Derived fields include:

- `subCategory`
- `category`

These are generated by:

- `extractAccessorySubCategory`
- `enrichAccessories`

## Filtering Flow

### UPS Filtering

UPS filtering is cascading:

```text
Brand -> Capacity -> Phase -> VDC -> Final UPS results
```

The related functions are:

- `getFilteredUpsBase`
- `getFilteredUpsCapacity`
- `getFilteredUpsPhase`
- `getFilteredUpsFinal`
- `renderUpsFilters`
- `renderUpsList`

When a higher-level filter changes, lower-level filters reset to `ALL` so invalid combinations are avoided.

### Battery Filtering

Battery filtering is:

```text
Type -> Brand -> AH -> Final battery results
```

The related functions are:

- `getFilteredBatteryBase`
- `getFilteredBatteryBrand`
- `getFilteredBatteryFinal`
- `renderBatteryFilters`
- `renderBatteryList`

### Accessory Filtering

Accessories are filtered by category:

```text
All Items / ATS / Charger / Rack / Service / SNMP / Other
```

The related functions are:

- `getFilteredAccessories`
- `renderAccessoryTabs`
- `renderAccessories`

## Cart And Quote Calculation

Items are added to the quote through `addToCart`.

Each cart item stores:

```js
{
  cartId: Number,
  productId: Number,
  name: String,
  rate: Number,
  warranty: String,
  category: String,
  qty: 1
}
```

Totals are calculated by:

- `getSubtotal`
- `getGST`
- `getTotal`
- `getItemCount`
- `updateHeader`

GST is calculated at 18%:

```js
Math.round(getSubtotal() * 0.18)
```

## Proposal And PDF Generation

The proposal is generated in a modal.

Important functions:

- `openModal`
- `closeModal`
- `buildProposalRows`
- `getProposalText`
- `copyProposal`
- `printProposal`

The PDF is created through the browser print dialog:

```js
window.print();
```

The user can choose **Save as PDF** from the print dialog.

## Print Layout

Print-specific behavior is controlled by the `@media print` block.

During printing:

- The main app shell is hidden.
- The proposal modal becomes the printable document.
- Customer details are displayed in a print-only block.
- On-screen-only controls are hidden.
- Tables and totals are restyled for clean A4 output.

The print-only customer fields are:

- `printCustomerName`
- `printCompanyName`
- `printContactNumber`
- `printSiteAddress`

They are populated inside `buildProposalRows` from the editable form fields:

- `customerName`
- `companyName`
- `contactNumber`
- `siteAddress`

## User Workflow

1. Open the app.
2. Filter UPS, battery, or accessory products.
3. Add required products to the quote.
4. Review the quote sidebar.
5. Click **Finalize Proposal**.
6. Enter customer details:
   - Customer name.
   - Company / organization.
   - Contact number.
   - Site address.
7. Click **Print / Save PDF**.
8. In the browser print dialog, select **Save as PDF**.

## Known Implementation Notes

- Product metadata is parsed from product names. If product naming changes significantly, the parser helper functions may need updates.
- Prices are stored directly in the front-end data arrays.
- The app does not persist cart data after page refresh.
- The app does not connect to a backend or database.
- The generated PDF depends on the browser print engine.

## Updating Product Data

To add or edit products, update the relevant array in `index.html`:

- UPS items: `UPS_DATA`
- Battery items: `BATTERY_DATA`
- Accessories: `ACCESSORY_DATA`

Keep product names consistent because filters are derived from the names.

For example, UPS names should include recognizable values such as:

- Brand
- kVA rating
- Phase
- VDC

Battery names should include recognizable values such as:

- Brand
- Battery type
- AH rating
- Voltage

## Troubleshooting

### PDF starts with blank pages

Check the print stylesheet. The main app shell should be hidden with `display: none` during print, and the modal should be the only printable layout.

### Customer details missing in PDF

Ensure the print-only customer block exists and is populated by `buildProposalRows`.

### Filters show unexpected results

Check product naming. Filter values are extracted from names using parser functions.

### Copy proposal does not work

Clipboard access may be blocked by the browser if the page is opened in an insecure or restricted context. Try serving the app from localhost.
